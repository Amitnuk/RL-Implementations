# DQN

## Introduction

### Motivation 

DQN, at its core, can be viewed as Q-learning with function approximation. As with any RL algorithm, the goal is to find a policy that maximizes the expected discounted cumulative future rewards. From the Bellman optimality equation, the optimal policy is defined as:


$$
\pi^{\ast}(s) = \arg\max_{a} Q^{\ast}(s, a)
$$

Where $$Q^{\ast}$$ represents the expected discounted cumulative future rewards obtained by taking action $$a$$ at state $$s$$ and following the optimal policy thereafter.

The RL objective is an optimization problem, but, unlike classical optimization problems, it breaks the usual key assumptions of stochastic optimization.
First, the data is not independent and identically distributed (i.i.d.). The trajectory generated by the agent is highly correlated as a consequence of learning through sequential decision-making.
Second, the target data is not only assumed to come from a fixed distribution but is also provided to guide the learning process; in RL, the targets are bootstrapped from the same object being optimized. As a consequence, the behavior distribution is constantly changing with each update.

Before DQN, many other approaches tried to mitigate this issue. [NFQI](https://github.com/Amitnuk/RL-Implementations/tree/main/deep_rl/value_based/nfqi), which can be seen as a precursor of DQN, tries to mitigate the mentioned problems by: (1) having the agent collect the trajectory (or trajectories), and (2) performing gradient descent on the collected data. This process is repeated throughout all episodes. Within an update, the targets are fixed, but they are recomputed for the next update. Thus, NFQI solves a different regression problem at each iteration. By choosing the size of the buffer containing the trajectories large enough, one can make the i.i.d. and fixed-distribution problems less pronounced. In NFQI, as with many methods before DQN, the convergence is not guaranteed, not only because of the usage of function approximation but also the inherent mathematical formulation of the algorithm. Moreover, its sampling strategy is not efficient.

DQN tackles the two problems mentioned by adding a mechanism to obtain (1) data that is approximately independent and more evenly distributed and (2) a slow moving target, effectively making RL problems more like a supervised learning problem.

### Background

DQN introduces two mechanisms, (1) experience replay and (2) target network. Together, they mitigate the aforementioned problems concerning i.i. and fixed target distribution. Let's start with experience replay then move to target network, which I believe to be the DQN groundbreaking idea.

* Experience replay : 

During the interactions in environment, the agent stores at each time step, $$e_{t}={s_{t}, a_{t}, r_{t}, s_{t+1}}$$ in the replay buffer *D*. 
Then, the Q-learning updates are done with mini-batches sampled uniformly from  *D*. This process has many advantages, i'll cite three. First it is sample efficient, beacause each transition can be used multiple times instead of being discarded after single update. Second it breaks the correlation in the data, resulting in lower variance of the updates. Third, it decouples the current policy behaviour with the batches selected for the updates, which can smmoth out the learning and reduce non-stationarity.

* Target network.

To improve the stability, a second network (target network) is introduced to generate the targets $$y_{j}=r_{j} + \gamma\max_{a'} Q(s',a'; \theta^-)$$. 
This technique mitigates the moving target issue. The target network $$Q(s', a'; \theta^-)$$ trails behind the main network $$Q(s,a; \theta)$$, where its weights are kept unchanged for every *M* steps. At the *Mth* step after performing gradient descent with respect to $$\theta$$, the second network is reset using the main network weights and the process repeats.
